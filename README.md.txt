YouTube 轉 Discord 自動化分流方案指南這套方案能夠自動監控指定的 YouTube 頻道，並根據內容類型（直播 vs 一般影片）分流通知到不同的 Discord 頻道。🛠️ GitHub Secrets 設定步驟為了確保 API Key 與 Webhook 資訊安全，請務必將變數存放在 GitHub Repository 的 Secrets 中：進入你的 GitHub Repository。點擊上方選單的 Settings。在左側選單中找到 Secrets and variables -> Actions。點擊 New repository secret 分別新增以下四個變數：變數名稱說明YOUTUBE_API_KEYGoogle Cloud Console 申請的 YouTube Data API v3 金鑰CHANNEL_ID要監控的 YouTube 頻道 ID (例如: UC...)DISCORD_WEBHOOK_A接收 一般影片 & Shorts 的 Discord 頻道 Webhook URLDISCORD_WEBHOOK_B接收 直播中 (Live) 通知 的 Discord 頻道 Webhook URL⚙️ 運作邏輯排程執行：預設每 15 分鐘執行一次。自動狀態維護：腳本執行完畢後若有新影片，會自動更新 last_notified.json 並 commit 回 GitHub 倉庫，避免重複通知。分流規則：liveBroadcastContent == 'live' -> Webhook B (直播)liveBroadcastContent == 'none' -> Webhook A (一般影片/Shorts)upcoming -> 忽略🚀 測試您可以點擊 GitHub 頁面的 Actions 頁籤，選擇 YouTube Monitor Automation，然後點擊右側的 Run workflow 手動觸發一次腳本來確認設定是否正確。
